# Dimensions: Padding, margin and block spacing

**Section:** Layout composition

## Transcript

### Introduction
Key dimension tools in the Block Editor: padding, margin, and block spacing.

### Accessing settings
Select block → Settings (top right) → Styles → Dimensions. If padding/margin not visible: three dots → enable.

### Definitions
- **Padding** — space inside a block, between content and border
- **Margin** — space outside the block, separating from other elements
- **Block spacing** — gap between nested blocks within a container (e.g. Columns, Group)

### Padding
- Horizontal + vertical sliders (0 to XXL)
- Custom values with units: px, %, rem, etc.
- **Unlink sides** — adjust top/left/bottom/right individually

### Margin
- Same controls as padding
- **Negative margins** possible (advanced setting)
- Use cases: add breathing room, space between header and content

### Block spacing
- Space between nested blocks in a container
- Set to 0 to unite elements (e.g. Gallery images)
- Adjust horizontal spacing between elements in Group

### Global defaults via Style Book
Set default padding, margin, block spacing for block types globally:
- Media: Media & Text block → add vertical padding
- Design: Buttons block → add horizontal + vertical padding
- Layout: main content area → vertical padding, block spacing

### Conclusion
Real-time visual feedback. Experiment with combinations — no custom CSS needed.

## Practical
1. Insert pattern → update Grid block spacing to XS
2. Insert pattern → update Columns padding to XX-Large
3. Add Buttons block "Learn more" → horizontal padding 10px, vertical 20px
4. Add Cover block twice → change bottom margin of first to Regular
