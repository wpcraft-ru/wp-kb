# Производительность SEO-плагинов: бенчмарки

> Дата сбора: 2026-05-03

## Методология тестирования

Тесты проводились на **идентичных staging-окружениях**:
- Один хостинг, одна версия WordPress, одна тема, один контент
- Замеры: execution time, memory usage, page load impact
- Инструменты: GTmetrix, Google PageSpeed Insights

## Ключевые выводы

### Скорость
- **The SEO Framework** — абсолютный лидер: в 12 раз быстрее Yoast/Rank Math
- **Slim SEO** — близко к TSF, минимальный footprint
- **Rank Math** — в 4 раза быстрее Yoast
- **SEOPress** — лёгкий, быстрый
- **Yoast SEO** — самый тяжёлый, но всё ещё Grade A

### Память
- **The SEO Framework:** 23.71 KB — на порядок меньше остальных
- **Rank Math:** +0.35 MB
- **Yoast:** +1.62 MB — в 4.6 раз больше Rank Math

### Размер кодовой базы
- **Yoast** в 1.6 раз больше Rank Math по строкам кода
- **Yoast** в 2.7 раз больше Rank Math по размеру папки

---

## Влияние на Core Web Vitals

Все плагины сохраняют Grade A на GTmetrix при стандартной установке. Однако:

- На **бюджетном хостинге** разница становится заметнее
- Yoast прибавляет 0.18s — на слабом хостинге это может быть критично
- The SEO Framework и Slim SEO практически незаметны
- Кеширование (WP Rocket) нивелирует разницу, но не полностью

|--------|-------------|----------------|---------|

---

## Почему производительность важна для SEO

1. **Google использует Core Web Vitals как фактор ранжирования**
2. На медленном хостинге каждый плагин на счету
3. Мобильная скорость критична (Mobile-First Indexing)
4. Пользовательский опыт: +1s задержки = -7% конверсий

---

## Рекомендация по производительности

1. **Если скорость — приоритет #1** → The SEO Framework
2. **Если нужен баланс скорость/функции** → Rank Math
3. **Если уже используете Yoast** → отключите неиспользуемые модули
4. **Всегда:** WP Rocket или другой кеширующий плагин
5. **Всегда:** отключайте неиспользуемые функции в любом SEO-плагине
