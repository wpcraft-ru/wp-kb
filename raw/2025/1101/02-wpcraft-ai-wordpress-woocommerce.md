## Составляющие AI Website Building

Инициатива включает четыре ключевых компонента, которые вместе создают инфраструктуру для внедрения ИИ в WordPress.

### Abilities

Abilities — это стандартизированные, обнаруживаемые, типизированные и исполняемые функции сайта на WordPress.

С версии WordPress 6.9 (ноябрь 2025) появился Abilities API — центральный реестр, куда плагины, темы и ядро регистрируют свои возможности (abilities).

Каждая Ability описывается четко:
- какие входные данные нужны (JSON Schema)
- что именно возвращается
- какие права доступа требуются
- документация для человека и для машины

Раньше функции были разбросаны по хукам, классам, REST API — их было сложно обнаружить и безопасно использовать извне. Теперь всё в одном унифицированном месте.

Примеры: создать пост, обновить настройки сайта, получить список пользователей, запустить специфическую бизнес-логику плагина.

### AI Agents

AI Agents — это продвинутые ИИ-системы, которые не просто отвечают текстом, а действуют автономно или полуавтономно: принимают задачу на естественном языке, планируют шаги, вызывают инструменты, читают данные, выполняют действия и возвращают результат.

В отличие от обычного чат-бота, агент может:
- несколько раз обращаться к внешним системам
- работать в цикле: рассуждение → действие → наблюдение → новое рассуждение
- использовать tool calling / function calling

В экосистеме WordPress (2026) это: Claude Desktop, Cursor, VS Code с GitHub Copilot / Continue / Cody и другие инструменты с поддержкой инструментов.

### Skills (Agent Skills)

Skills — это готовые пакеты промптов + инструментов, которые добавляются в арсенал AI-агента, чтобы он лучше понимал и работал именно с WordPress.

Примеры:
- wp-playground skill — автоматическое тестирование кода в WordPress Playground
- theme-development skill
- plugin-creation skill
- seo-optimization skill

Живут в публичных репозиториях, например github.com/WordPress/agent-skills.

### MCP (Model Context Protocol)

MCP — это открытый протокол (стандарт), который позволяет AI-агентам безопасно и унифицировано подключаться к внешним системам (включая WordPress), обнаруживать доступные инструменты и данные, а потом их вызывать.

Основные функции MCP:
- обнаружение возможностей (discover)
- передача tools, resources, prompts
- авторизация (чаще всего OAuth 2.1)
- транспорт (HTTP, STDIO и др.)

В WordPress два основных способа (2026):
- WordPress.com — встроенный MCP-сервер (с 2025)
- Self-hosted WordPress — WordPress MCP Adapter (февраль 2026)

## Чего ожидать владельцам сайтов

- Упрощённое внедрение ИИ: плагины типа AI Experiments Plugin
- Выбор провайдера ИИ и единое управление API-ключами
- Автоматизация задач: ИИ-ассистенты управляют сайтом через диалоговый интерфейс
- Интеграция с Gutenberg
- Интеграция с хостингами (например, amazee.ai)
- Перспективы к WordPress 7.0 (2026): ИИ-функции в Site Admin, Gutenberg и медиабиблиотеке
