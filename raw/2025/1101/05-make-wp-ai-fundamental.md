## WordPress Fundamentals

Автор предлагает мысленный эксперимент: представьте WordPress без базы данных. База данных настолько фундаментальна, что никто не проверяет её наличие перед использованием `get_post_meta()`. Плагины строятся на предположении, что БД существует.

## AI как фундамент

Что если ИИ станет таким же фундаментальным, как база данных? Что если от конечного пользователя до разработчика плагинов все могли бы принимать как должное наличие доступной AI-модели?

Не надо думать об ИИ как о функции (чат-интерфейс). Это нечто большее. Например: кнопка «Сгенерировать alt-текст» для изображения — это AI-powered функция, но пользователь не думает «я использую ИИ».

## AI функции в масштабе WordPress

Самое невероятное в WordPress — это экосистема. Каждый день разработчики совершенствуют плагины. Представьте, если бы каждый разработчик имел доступ к AI без необходимости разбираться в сложностях интеграции:

```php
$image = Ai_Client::prompt('Create an image that beautifully reflects this post content')
  ->with_text($post_content)
  ->generate_image();
```

Это реальный код из [WP AI Client](https://github.com/wordpress/wp-ai-client), предложенный для WordPress 7.0!

**Ключевая идея:** AI должен стать выигрышем для всей экосистемы. Принятие AI экосистемой — самая необходимая цель.

## Стратегия внедрения

Главный вызов — как дать всем доступ к AI. Сейчас интеграция с LLM требует либо собственного сервиса (как Elementor's Angie), либо от пользователя — сгенерировать ключ OpenAI.

**Решение:** пусть хостеры предоставляют AI так же, как базы данных. Если бы БД была опциональной в WordPress, хостеры с БД имели бы огромное преимущество. Аналогично — хостеры, включающие AI-модель в хостинг-план.

### Обратная совместимость

WP AI Client имеет систему поиска лучшей модели для промпта и методы проверки доступности модели. Разработчик может использовать проверки для условного отображения функций («Generate Alt text» показывается только если есть поддерживающая модель).

## Где мы сейчас

Путешествие уже началось:
- Сформирована AI Team в WordPress
- Создан AI Building Block
- Abilities API (WP 6.9)
- WP AI Client (предложен для WP 7.0)
- MCP Adapter
- Workflows API (в разработке) — цепочки Abilities в мощные потоки

### Разработчикам
Использовать Abilities API, WP AI Client, MCP Adapter. Workflows API позволит связывать Ability в цепочки. Пример: публикация поста → AI-summary → AI-generated email → уведомление в Slack.

### Хостерам
Критически важны для распространения AI-моделей. Включая AI в хостинг-планы, WordPress становится ещё мощнее.
