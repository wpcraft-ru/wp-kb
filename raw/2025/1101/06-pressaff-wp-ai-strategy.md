# Стратегия WordPress в эпоху ИИ: ставка на инфраструктуру, а не интерфейсы

**Источник:** https://pressaff.com/articles/wordpress-is-changing-its-approach-to-ai/
**Дата сбора:** 2026-05-01

---

Пока рынок соревнуется в количестве ИИ-ассистентов, WordPress пошел другим путем — начал перестраивать фундамент платформы. Вместо одного умного помощника появляется целая архитектура для агентов, автоматизации и безопасной интеграции ИИ.

## Почему WordPress не стал делать «свой ChatGPT»

В 2025 — начале 2026 года WordPress сделал самый нетипичный ход. Вместо выпуска собственного «умного ассистента» платформа пошла вглубь — к архитектуре.

**Ключевая идея:** любой ИИ-агент может управлять WordPress, если он знает правила игры. Неважно, Claude, ChatGPT, Copilot или Cursor — платформа не делает ставку на «своего» агента.

## Основные компоненты ИИ-экосистемы WordPress

### Abilities API

Встроен в ядро WordPress 6.9. Единый реестр всех возможностей сайта — неважно, относятся они к ядру, теме или стороннему плагину.

Каждая ability имеет:
- Название (например, «получить данные сайта»)
- Описание входных данных
- Описание результата
- Проверку прав доступа
- Логику выполнения

В базовой поставке WP 6.9: получение информации о сайте, данные о пользователе, техническая информация о среде.

### MCP Adapter

Объединяет Abilities API с ИИ через Model Context Protocol. Превращает внутренние «способности» сайта в формат для внешних инструментов.

Три базовых примитива MCP:
- **tools** — исполняемые действия (создать пост, обновить запись)
- **resources** — пассивные источники данных (логи, конфигурации)
- **prompts** — заготовленные шаблоны рабочих процессов

Поддерживает: STDIO для локальной разработки, HTTP для публичных сайтов. Протестирован с Claude Desktop, Claude Code, Cursor, VS Code.

### WP AI Client

Инструмент для работы с генеративным ИИ внутри WordPress. Позволяет плагинам обращаться к разным провайдерам (OpenAI, Anthropic, Google) через единый API.

- Использует HTTP API WordPress
- Интегрируется с админкой для хранения API-ключей
- Автоматический выбор подходящей модели
- Приоритетные модели — WordPress пробует первую доступную из списка

**Для разработчиков:** снимает боль мультивендорности.
**Для владельцев:** свобода выбора провайдера без замены плагинов.

### Agent Skills

Open-source библиотека навыков для ИИ-агентов. Решает «хронические болезни» ИИ при работе с WordPress:
- Использование устаревших паттернов
- Игнорирование block.json, депрекаций и блочных тем
- Пропуск базовых проверок безопасности

Каждый skill содержит: триггеры (когда применять), пошаговые инструкции, чек-листы, примеры правильных и неправильных решений.

## Вывод

WordPress не становится ещё одной витриной с ИИ — он превращается в платформу, с которой могут работать все будущие агенты. Сначала правила, потом подключение ИИ.
