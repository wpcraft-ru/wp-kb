## Что такое плагин

Плагин = функциональность. Плагины расширяют WordPress, не трогая ядро.
Устанавливаются из официального репозитория (`Плагины → Добавить новый`) или вручную через ZIP.

## Что проверять перед установкой

1. **Частота обновлений** — последнее обновление не старше 6–12 месяцев.
2. **Совместимость** — метка «Проверено с» должна быть близка к текущей версии WordPress.
3. **Репутация автора** — количество активных установок и оценки пользователей.
4. **Дублирование** — плагин не должен перекрывать функциональность уже установленных решений.

## Жизненный цикл плагина

```
Установка → Активация → Настройка → Обновления → Деактивация → Удаление
```

При деактивации данные обычно сохраняются. При удалении — зависит от плагина (часть оставляет таблицы в БД).

## PHP: проверка активности плагина

```php
if ( is_plugin_active( 'woocommerce/woocommerce.php' ) ) {
    // WooCommerce активен — можно использовать его функции
}
```

Что делает код:
- Проверяет, активирован ли конкретный плагин по его пути `plugin-folder/main-file.php`.
- Функция доступна только в admin-контексте; для фронтенда используйте `function_exists()`.

## Безопасные практики

- Устанавливать плагины только из официального репозитория WordPress или от проверенных авторов.
- Не держать деактивированные плагины — они всё равно сканируются и могут содержать уязвимости.
- Делать резервную копию перед обновлением значимых плагинов.
