## Где находятся настройки

**Настройки → Обсуждение** (`/wp-admin/options-discussion.php`).

## Основные группы настроек

### Настройки записи по умолчанию (Default post settings)

| Настройка | Что делает |
|---|---|
| Оповещать блоги о новых записях | Пингует другие сайты при публикации |
| Принимать пинги и уведомления | Разрешает другим сайтам ссылаться |
| Разрешить комментарии | Включает форму комментариев для новых записей |

> Эти настройки — значения по умолчанию. Их можно переопределить для каждой конкретной записи.

### Настройки комментариев (Other comment settings)

- **Автор должен указать имя и email** — обязательные поля для комментирования.
- **Пользователи должны быть зарегистрированы** — только авторизованные могут оставлять комментарии.
- **Закрывать комментарии через N дней** — автоматическая блокировка старых записей.
- **Разбивка на страницы** — пагинация для длинных обсуждений (20–50 комментариев на страницу).
- **Сначала старые / новые** — порядок показа комментариев.

### Уведомления по email

- **Email автору при новом комментарии**.
- **Email автору при одобрении комментария**.

### Модерация (Before a comment appears)

- **Комментарий должен быть одобрен вручную** — каждый комментарий ждёт модератора.
- **Автор должен иметь одобренный комментарий** — после первого одобрения следующие публикуются автоматически.

### Список модерации (Comment Moderation)

Поле для ключевых слов. Комментарии с этими словами в тексте, URL, email, IP или User Agent автоматически попадают на модерацию (или в спам).

### Запрещённые ключи (Disallowed Comment Keys)

Комментарии с этими словами автоматически отправляются в корзину.

### Аватары (Avatars)

- **Показывать аватары** — вкл/выкл.
- **Рейтинг** — G (для всех), PG, R, X.
- **Аватар по умолчанию** — Mystery Person, Gravatar Logo, Identicon, Wavatar и др.

## Практические рекомендации

1. **Для обычного блога:** включите комментарии, обязательные имя+email, ручную модерацию для комментариев с 1+ ссылок.
2. **Для бизнес-сайта:** отключите комментарии вообще, если они не нужны.
3. **Для сообщества:** включите регистрацию, оставьте комментарии открытыми, но настройте список модерации.

## Антиспам

На self-hosted WordPress антиспам настраивается через плагины:

- **Akismet Anti-Spam** — стандартный плагин, требует API-ключ.
- **Antispam Bee** — бесплатный opensource-вариант без ключей.
- **CleanTalk** — облачный сервис с защитой от спама в формах, комментариях, регистрациях.
