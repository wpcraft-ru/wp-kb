## Что такое SEO

SEO (Search Engine Optimization, поисковая оптимизация) — набор практик и методов для повышения видимости сайта в результатах поиска и улучшения его рейтинга. Цель — привлечение органического (бесплатного) трафика через повышение авторитета сайта и релевантности контента для поисковых алгоритмов.

## Пять основных типов SEO

### 1. On-Page SEO (внутренняя оптимизация)

Оптимизация элементов на страницах сайта, которыми вы управляете напрямую:

- **Ключевые слова** — исследование и встраивание в title, заголовки H1-H3, мета-описания
- **Качественный контент** — полезный, экспертный, отвечающий на запросы
- **Структура страницы** — правильная иерархия заголовков (H1 → H2 → H3)
- **Оптимизация изображений** — сжатие, современные форматы (WebP, AVIF), alt-текст
- **Внутренние ссылки** — перелинковка между страницами сайта
- **URL-структура** — читаемые, короткие URL с ключевыми словами
- **CTA** — призывы к действию

### 2. Off-Page SEO (внешняя оптимизация)

Действия за пределами сайта для повышения авторитета:

- **Линкбилдинг** — получение качественных обратных ссылок с авторитетных сайтов
- **Упоминания бренда** — в интернете, соцсетях, на форумах
- **Гостевой блогинг** — публикации на сторонних площадках
- **Социальные сети** — продвижение и вовлечение аудитории
- **Influencer-маркетинг** — работа с лидерами мнений
- **Цифровой PR** — управление репутацией

### 3. Local SEO (локальное SEO)

Оптимизация для локального поиска (например, «ремонт телефонов Москва»):

- **Google Business Profile** — создание и оптимизация профиля компании
- **Яндекс.Бизнес** — аналог для Яндекса
- **NAP** — консистентность названия, адреса и телефона на всех площадках
- **Локальные ключевые слова** — геопривязка запросов
- **Отзывы** — работа с отзывами клиентов

### 4. International SEO (международное SEO)

Оптимизация для разных стран и языков:

- **URL-структура** — ccTLD (домены стран), субдомены, подпапки с geo-привязкой
- **Hreflang-теги** — указание языка и региона для страниц
- **Локализация контента** — адаптация под культуру и запросы региона

### 5. Technical SEO (техническая оптимизация)

Фундамент сайта для корректного сканирования и индексации:

- **Скорость загрузки** — оптимизация TTFB, рендеринга
- **Core Web Vitals** — LCP, INP, CLS
- **Мобильная адаптивность** — mobile-first индексация Google
- **Архитектура сайта** — логичная структура и навигация
- **HTTPS** — SSL-сертификат обязателен
- **XML Sitemap и robots.txt** — управление индексацией
- **Структурированные данные (Schema Markup)** — JSON-LD разметка
- **Исправление ошибок** — битые ссылки, дубли, soft 404

## SEO в 2026 году: ключевые тренды

### Интент вместо точных вхождений

Google и Яндекс ранжируют по **намерению пользователя (интенту)**, а не по точному совпадению слов. Шаблонный текст не работает — нужны реальные кейсы, цифры, экспертиза.

### Core Web Vitals: INP вместо FID

С марта 2024 года оценивается **INP (Interaction to Next Paint)** — отклик интерфейса на всей сессии, а не только первого взаимодействия:

- До 200 мс — хорошо
- 200–500 мс — требуется улучшение
- Выше 500 мс — проблемная зона

Причины ухудшения: тяжёлый JavaScript, отсутствие code splitting, сложные UI-библиотеки.

### E-E-A-T

Модель оценки качества: **Experience, Expertise, Authoritativeness, Trustworthiness**. Контент выигрывает, если указан автор, есть реальный опыт, конкретные данные, бизнес прозрачен. Критично для YMYL-тем (здоровье, финансы).

### Нейроответы в выдаче

Поиск всё чаще формирует краткие ответы прямо в выдаче без перехода на сайт. Чтобы попасть в такие блоки:
- Давать суть в начале (BLUF — Bottom Line Up Front)
- Конкретные выводы, без воды
- Чёткая структура и факты

### JavaScript и индексация

Поисковики обрабатывают JS, но с задержкой. Критический контент должен быть доступен в HTML без JS. SSR или SSG снижают риски.

### Crawl Budget для крупных сайтов

Актуально для e-commerce и сайтов с фильтрами. Решения: каноникализация фильтров, закрытие служебных параметров, контроль через Search Console.

### Soft 404 и IndexNow

- **Soft 404** — если товар удалён навсегда, отдавать 404 или 410; если временно отсутствует — страница должна оставаться полезной
- **IndexNow** — протокол ускоряет переобход для Яндекса и Bing, не влияет на ранжирование

## Алгоритмы Google и Яндекса

### Google

| Алгоритм | Назначение |
|----------|-----------|
| **PageRank** | Оценка значимости сайта по входящим ссылкам |
| **BERT / MUM** | Понимание естественного языка и длинных запросов |
| **Helpful Content System** | Фильтр низкокачественного контента |
| **E-A-T** | Оценка экспертизы, авторитетности и доверия |

### Яндекс

| Алгоритм | Назначение |
|----------|-----------|
| **YATI** | Нейросетевой индекс, понимание смысла запроса |
| **Матрикснет** | Машинное обучение для ранжирования |
| **Королёв** | Нейросетевой алгоритм для смысловой оценки контента |

### Ключевые отличия

| Параметр | Google | Яндекс |
|----------|--------|--------|
| Язык | Лучше понимает естественный язык | Лучше учитывает региональные запросы |
| JS | Обрабатывает с задержкой | Проблемы со сканированием JS |
| Коммерческие факторы | Меньше влияния | Важны: контакты, цены, отзывы |
| IndexNow | Тестирует | Внедрён глубже |

## Типичные ошибки SEO

- Массовая генерация ИИ-контента без экспертной доработки
- Индексация технического мусора (фильтры, дубли)
- Игнорирование Core Web Vitals
- Сломанный hreflang
- Отсутствие регулярного аудита индексации
