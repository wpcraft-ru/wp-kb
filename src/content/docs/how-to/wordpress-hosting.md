## Зачем нужен хостинг

Хостинг — это место на сервере, где хранятся все файлы, изображения и контент вашего сайта. Без хостинга сайт недоступен в интернете. В отличие от WordPress.com с его встроенным managed-хостингом, при использовании WordPress.org вы выбираете хостинг самостоятельно.

## Типы хостинга для WordPress

### Shared (виртуальный) хостинг
Самый доступный вариант. Несколько сайтов живут на одном сервере и делят ресурсы.

**Плюсы:** низкая цена (150-500 ₽/мес), простота, автоустановка WordPress.  
**Минусы:** ограниченные ресурсы, производительность зависит от соседей по серверу.  
**Для кого:** блоги, визитки, лендинги с небольшим трафиком.

### VPS (Virtual Private Server)
Виртуальный выделенный сервер с гарантированными ресурсами.

**Плюсы:** изолированная среда, высокая производительность, root-доступ.  
**Минусы:** требует навыков администрирования, дороже shared (500-3000 ₽/мес).  
**Для кого:** растущие проекты, магазины, сайты с высоким трафиком.

### Облачный хостинг
Сайт работает на кластере серверов, легко масштабируется.

**Плюсы:** масштабируемость, оплата по факту использования, высокая доступность.  
**Минусы:** сложная ценовая модель, требует технических знаний.  
**Для кого:** высоконагруженные проекты, SaaS.

### Managed WordPress Hosting
Хостинг, оптимизированный специально под WordPress. Провайдер берёт на себя обновления, бэкапы, безопасность.

**Плюсы:** не нужно думать о технических деталях, автообновления, экспертная поддержка.  
**Минусы:** дороже обычного хостинга, ограничения по плагинам.  
**Для кого:** те, кто хочет «как на WordPress.com, но своё».

## Критерии выбора хостинга

- **PHP-версия:** минимум 8.0, лучше 8.2+
- **MySQL/MariaDB:** поддержка актуальных версий
- **SSL:** бесплатный Let's Encrypt
- **Бэкапы:** ежедневные автоматические
- **Техподдержка:** русскоязычная 24/7
- **Кэширование:** встроенное (LiteSpeed, Redis) или совместимость с плагинами кэша
- **Локация серверов:** для РФ-аудитории — серверы в России/Европе

## Что обычно включено в managed-хостинг и как это сделать самому

| Функция WordPress.com Managed | Self-hosted альтернатива |
|---|---|
| Автообновления WordPress | Встроено в WordPress (настроить автообновления) |
| Автообновления PHP | Ручное переключение версии в панели хостинга |
| Бэкапы с возможностью восстановления | Плагин UpdraftPlus + облачное хранилище |
| Безопасность (защита от взлома, malware scan) | Плагин Wordfence + регулярные обновления |
| Встроенный CDN | Cloudflare (бесплатный тариф) |
| SSL-сертификаты | Let's Encrypt (бесплатно, есть на большинстве хостингов) |
| Статистика и аналитика | Google Analytics + плагин Site Kit |
| Email-хостинг | Яндекс.Почта для домена, Mail.ru для бизнеса |

## PHP пример: определение окружения для конфигурации

```php
// wp-config.php — настройки под конкретный хостинг
if (defined('WP_ENV') && WP_ENV === 'production') {
    define('WP_DEBUG', false);
    define('WP_CACHE', true);
} else {
    define('WP_DEBUG', true);
    define('WP_CACHE', false);
}
```

Что делает код:
- Разделяет конфигурацию production/staging.
- Отключает отладку и включает кэш на боевом сервере.
