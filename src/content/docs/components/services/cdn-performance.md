## Обзор

CDN (Content Delivery Network) — сеть серверов, распределённых географически, которая кэширует статические файлы (CSS, JS, изображения) и отдаёт их посетителю с ближайшего узла. Это снижает latency и нагрузку на origin-сервер.

Для WordPress CDN особенно важен: типичный сайт содержит десятки изображений, скриптов и стилей на каждой странице. Правильно настроенный CDN может снизить LCP на 30–70%.

## Международные сервисы

### Cloudflare

Глобальный CDN с 250+ edge-локациями. Бесплатный тариф включает SSL, базовый DDoS и кэширование статики. Ключевая фича — **APO (Automatic Platform Optimization)** на платных тарифах: кэширует HTML на edge, снижая TTFB на 70%+.

- **Плагин:** официальный Cloudflare для WordPress — авто-интеграция DNS, сброс кэша при обновлении контента
- **Рекомендация:** использовать вместе с WP Rocket или LiteSpeed Cache
- **Цены:** Free → Pro $20/мес → Business $200/мес
- **Для кого:** сайты с глобальным трафиком, визуальным контентом, всплесками трафика

### Bunny.net

Pay-as-you-go CDN: ~$0.01/GB. 100+ PoP. Отличное соотношение цена/качество для малых и средних сайтов.

- **Плагин:** Bunny WordPress CDN + интеграция с WP Rocket
- **Плюсы:** очень дёшево (~$50 за 5 TB), простой старт (DNS + плагин = минуты)
- **Минусы:** нет full-page кэширования на уровне CDN, нужен отдельный кэш-плагин
- **Для кого:** блоги, портфолио, small business

### NitroPack

All-in-one облачный сервис: кэширование + оптимизация изображений + минификация + lazy-loading + CDN — всё в одном.

- **Результат:** PageSpeed часто 90+ сразу после активации
- **Заменяет:** кэш-плагин + оптимизатор изображений + CDN
- **Минусы:** vendor lock-in, нет оптимизации БД, feature-locking на дешёвых тарифах
- **Цены:** Free (5K views) → Business $21/мес → Growth $51/мес → Scale $176/мес
- **Для кого:** те, кому нужен результат «из коробки» без сложной настройки

## Российские сервисы

### Yandex Cloud CDN

37 PoP в России + СНГ. Отличные TTFB для РФ/СНГ аудитории. Гибкая настройка TTL и GZIP-сжатия. Интеграция через CNAME + настройки кэширующего плагина.

- **Плюсы:** низкие задержки для РФ/СНГ, интеграция с экосистемой Yandex Cloud
- **Минусы:** меньше PoP для глобальной аудитории, нет APO-подобного HTML-кэша
- **Для кого:** сайты с аудиторией РФ, Казахстан, Узбекистан

### Selectel CDN

Облачные серверы + S3-хранилище + CDN. Плагин **Selectel Storage Upload** синхронизирует медиафайлы в облако — можно хранить файлы только там, без дублирования на сервере. CDN работает через Akamai.

- **Схема:** облачный сервер → WordPress → синхронизация в S3 → раздача через CDN
- **Плюсы:** снижение нагрузки на сервер, отказоустойчивость
- **Минусы:** сложнее старт, нужна настройка сервера и DNS

### VK Cloud CDN

Готовый WordPress в «магазине приложений», собственная CDN-сеть VK, S3-хранилище, HTTP/3, DDoS-защита.

- **Интеграция:** через W3 Total Cache / WP Super Cache (Generic Mirror)
- **Плюсы:** российская инфраструктура, HTTP/3, гибкое масштабирование
- **Минусы:** требует Linux-администрирования, не для новичков
- **Для кого:** средние проекты и магазины с аудиторией РФ/СНГ

## Как выбрать

1. **Глобальная аудитория** → Cloudflare (бесплатный старт, APO для роста)
2. **Бюджетный CDN для статики** → Bunny.net + WP Rocket
3. **Set-and-forget** → NitroPack
4. **РФ/СНГ аудитория** → Yandex Cloud CDN или Selectel
5. **Нужен контроль над инфраструктурой** → VK Cloud

Независимо от выбора CDN, стандартная рекомендация: кэширующий плагин (WP Rocket, LiteSpeed Cache, W3 Total Cache) + CDN для статики. Подробнее о кэшировании — [Кэширование WordPress](../../performance/cache.md).
